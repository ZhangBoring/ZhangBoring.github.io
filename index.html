Redis缓存问题(击穿、穿透、雪崩)及解决方案

Posted on 2020-09-18 Redis 缓存问题

缓存雪崩

介绍

大量缓存数据集体失效，导致大量请求直接打到数据库，造成数据库压力过大导致响应不及时，甚至会使数据库宕机进而整个系统的崩溃。

解决方案

  1. 在设置key的时候，随机生成失效时间
  2. 在基于集群的场景下，可以将热点key平均分散到不同节点上，可以分散雪崩的压力
  3. 不设置失效时间(永久生效)
  4. 定时任务刷新key，在key即将过期时刷新失效时间

缓存穿透

缓存穿透和缓存击穿这两个概念很容易混淆。

介绍

个人认为缓存穿透的重点在于透，目的就是透过Redis直接访问数据库，是一种恶意攻击行为，方法是获取明显不存在的数据，如：自增主键(0开始)下请求获取id小于0的数据。

解决方案

  1. 将空数据放到Redis里，只针对某一个参数
  2. 拉黑请求IP，不影响恶意用户切换IP
  3. 参数过滤，校验参数合法性
  4. 布隆过滤器

缓存击穿

介绍

某一个热点数据失效，导致大量请求直接访问数据库(与穿透不同的是，此处请求的资源是存在的)。

解决方案

  1. 热点key永不过期
  2. 分布式锁(分布式: ZoKeeper或Redis)/互斥锁(单体)
    • 保证只有一个请求去访问数据库
    • 该请求将数据添加到缓存中
    • 其他请求从缓存中获取并返回

  • Circuit Breakers 断路器
  • Global locks 全局锁
  • Leadership election and cluster state 选举与集群状态管理

API接口及实现类

类型 名称 描述
interface Executor 最上层的接口，定义了执行任务的方法execute(Runnable)
interface ExecutorService 扩展Executor接口，添加了Callable支持，定义了任务提交方法submit及关闭方法shutdown
interface ScheduledExecutorService 扩展了ExecutorService接口，增加定时任务相关方法
class ThreadPoolExecutor 标准的线程池实现类

定时任务接口中定义的两个周期执行方法区别：

  • scheduleAtFixedRate方法：在给定延迟后开始执行，如果遇到异常，则禁止后续执行。若执行时间大于执行周期，则当前任务执行完成后，马上执行下次任务。
  • scheduleWithFixedDelay：给定延迟后开始执行，每次任务执行完成后，经过执行周期时间，再执行下次任务。

学习Redis-数据类型

Posted on 2020-08-10 Edited on 2020-09-16 Redis

介绍

学习Redis的键、值支持的数据类型。

  • Redis的key是二进制安全的，所以可以使用任意二进制序列作为key值，包括空字符串。有以下规则：
    • 长度不能过长。长度过长不仅会消耗内存，也会增加查找时的计算成本；
    • 键名要有意义、高可读性，不要有特殊字符(如空格、换行、引号、转移字符等)，且统一命名模式，如：“业务名:表名:id”；
  • Redis的值支持以下类型：
    1. 字符串(strings)
    2. 散列(hashes)：由field和关联的value组成的map，field和value都是字符串类型；
    3. 列表(lists)：类似链表(linked lists)；
    4. 集合(sets)：不重复、无序的字符串集合；
    5. 有序集合(sorted sets)：类似sets，但是每一个元素都关联着一个score，元素通过score进行排序；
    6. Bit arrays(simply bitmaps)：通过特殊命令，可以像bits数组一样处理字符串：设置或清楚单独的bits，统计所有设为1的bits，找到最先倍设为1/0的bits，等等；
    7. HyperLogLogs：统计一个set中元素技术概率性的数据结构；
    8. Streams[5.0新特性]

适用场景

数据类型 特点 适用场景
string 简单的k-v结构 简单的字符串场景，如计数等
list 1. 元素类型为String的双向链表结构；2. redis也提供了阻塞获取的api 1. 存储登录人id等；2. 消息队列
hash key:field-value，映射表 存储对象，如用户实体信息
set 单一string值的无序集合，不同集合集合间可以方便地求交集并集 将用户的关注列表放到set中，取交集的共同关注等
sorted set 带权重参数的set 排行榜
