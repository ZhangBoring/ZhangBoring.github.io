Java集合-HashMap

Posted on 2020-07-24 Edited on 2020-08-28 Java

数据结构

HashMap底层是由数组+链表/红黑树构成的。
在存储数据时，先计算key的hash值，通过hash值来确定数据在数组中存放的位置；如果出现hash冲突，则数据以链表的方式存储；链表长度过长，转换成红黑树。

源码学习

tableSizeFor(int cap)

在使用HashMap(int initialCapacity, float loadFactor)构造方法创建对象时，构造方法中会执行tableSizeFor(initialCapacity)计算数组长度，以保证数组长度为2的整数次。

  • cap - 1：保证结果是距离给定值最近的2的幂次。例如下表

    操作 cap-1 cap
    cap = 8(1000) 7(111) 8(1000)
    n |= n >>> 1 7(111) 12(1100)
    n |= n >>> 2 7(111) 15(1111)
    n |= n >>> 4 7(111) 15(1111)
    … … …
    return n+1 8(1000) 16(10000)

  • 数组长度为2的幂次的原因：在通过与运算计算数据下标(index=hash&(length-1))时,可以充分利用每一位。假如长度不为2的幂次，如：hash&11100这种情况(长度为29)，每次与运算结果后两位都是0，浪费数组空间。

扩容 resize()

resize()方法：如果hashMap对象没有倍初始化，则实际初始化数组；或者元素数量达到阈值，进行扩容。

插入 put(K key, V value)

在插入元素时，如果数组为空，调用扩容方法初始化数组；计算hash定位下标，插入新元素；如果是链表，判断是否需要树化；判断是否需要扩容。

  • 链表树化：在put新元素时，如果链表长度大于等于8，并且数组长度大于等于64，则将该链表转换成红黑树。若数组长度小于64，发生hash冲突的概率比较大，应先进行扩容。

计算hash hash(Object key)

int有32位，将对象的hashCode右移位16位，进行异或运算。低16位与高16位进行异或运算，增加hash的随机型。
