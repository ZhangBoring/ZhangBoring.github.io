Java-线程池

Posted on 2020-08-22 Edited on 2020-11-01 Java

API接口及实现类

类型 名称 描述
interface Executor 最上层的接口，定义了执行任务的方法execute(Runnable)
interface ExecutorService 扩展Executor接口，添加了Callable支持，定义了任务提交方法submit及关闭方法shutdown
interface ScheduledExecutorService 扩展了ExecutorService接口，增加定时任务相关方法
class ThreadPoolExecutor 标准的线程池实现类
class ScheduledThreadPoolExecutor 继承ThreadPoolExecutor，实现ScheduledExecutorService，实现定时任务方法

定时任务接口中定义的两个周期执行方法区别：

  • scheduleAtFixedRate方法：在给定延迟后开始执行，如果遇到异常，则禁止后续执行。若执行时间大于执行周期，则当前任务执行完成后，马上执行下次任务。
  • scheduleWithFixedDelay：给定延迟后开始执行，每次任务执行完成后，经过执行周期时间，再执行下次任务。

      线程池创建新线程的规则：

      1. 如果正在运行的线程数小于corePoolSize时，即使其他工作线程处于空闲状态，也会创建一个新线程来处理请求；
      2. 如果运行的线程数大于corePoolSize但小于maximumPoolSize时，则仅在队列已满时才创建新线程；
      3. 答案：由于创建线程池时指定的线程池为无界队列，所以永远不会创建新线程，而是一直复用3个核心线程来处理任务。

后记：拒绝策略

JDK提供拒绝策略演示Demo

介绍

  • 在JDK中线程池类ThreadPoolExecutor中有预定义四种拒绝策略：
  • 描述：
    名称 描述
    CallerRunsPolicy 直接在调用当前线程池的线程中执行拒绝添加的任务，如果执行线程已关闭，则丢弃任务。问题：会阻塞主线程。
    AbortPolicy 丢弃任务并抛出RejectedExecutionException异常。线程池的默认拒绝策略，可以及时发现系统出现的异常。
    DiscardPolicy 静默丢弃添加失败的任务。无法及时发现系统的异常状态。
    DiscardOldestPolicy 丢弃队头的任务(最早的未处理请求)，然后重试执行(e.execute(r))。如果线程池被关闭，则丢弃任务。
