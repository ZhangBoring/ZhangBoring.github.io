Java 集合框架

Posted on 2021-01-21 Edited on 2021-02-20 Java 集合

Java集合类图

可以看到，类图整体分为四部门，分别是：Map、Set、List和角落里的的工具类的Collections和Arrays。

List部分

关于ArrayList和LinkedList的特点和用法，在之前的文章中有介绍：Java集合-List

这里说明一下涉及到的几个接口。

RandomAccess接口

标记接口，该接口没有任何方法，仅用于标记List的实现类以指示他们支持随机访问。可以看到ArrayList实现了该接口，但是LinkedList未实现该接口，也就是说ArrayList支持随机访问。

Deque接口

双端队列接口，LinkedList实现了该接口，即LinkedList是通过双向链表实现的。

在Deque中有三个方法，E pop()、void push(E e)和E peek()，其中pop和push是为栈定义的方法。
所以在JDK中没有栈专门的接口，只有一个Stack类。但是JDK推荐使用Deque的实现类来作为栈这种数据结构，Stack的源码注释中是这么说的：

Map部分

Map相关实现类中比较重要的有HashMap、TreeMap和LinkedHashMap。

HashMap

在之前的文章中，有阅读分析过HashMap的源码：Java集合-HashMap

LinkedHashMap

LinkedHashMap继承自HashMap，并且子类Entry节点类扩展了HashMap的基础节点类，由原先的单项链表扩展为双向链表。

LinkedHashMap是有序的，在默认情况下构造的list对象accessOrder是为false的，是根据put的顺序存储。

Set部分

不包含重复元素的集合。集合中不存在满足e1.equals(e2)的元素对，并且最多只能有一个null元素。

HashSet

底层是一个HashMap的实例。不保证元素的顺序。map元素的key值是set中的额元素值，value值是一个final对象：

HashSet不是线程安全的。如果多个线程同时访问同一个集合实例，并且至少有一个线程做修改操作，则必须在外部进行同步：
Set s = Collections.synchronizedSet(new HashSet(...));

LinkedHashSet

LinkedHashSet继承自HashSet，区别是底层使用LinkedHashMap，元素的遍历顺序和插入顺序是一致的。

  1. 需要相比于前一个方法多传一个比较器(Comparator)

Arrays

该类包含用于操纵数组的各种方法，例如排序和搜索。如果指定的数组引用为null，则除非声明另有说明，否则此类中的所有方法均抛出NullPointerException。

sort 和 parallelSort

  1. sort 方法是对数组按照升序排序

  2. parallelSort 方法是先将数组分解为子数组，这些子数组本身经过排序后在进行合并。当子数组的长度达到最小粒度时，将使用适当的Arrays.sort方法对子数组进行排序。

parallelPrefix

并行地计算数组中的每一个数据。示例

binarySearch: 二叉搜索

equals: 判断两个数组是否相等

要求两个数组长度必须相等，并且需要两个数组同一个index下的元素相等（非基本数据类型会调用Objects.equals方法比较）。

fill: 填充

copyOf: 复制

复制指定的数组，截断或使用null填充（如果需要），以便副本具有指定的长度。对于在原始数组和副本中均有效的所有索引，两个数组将包含相同的值。对于副本中有效但原始索引无效的任何索引，副本将包含null。当且仅当指定长度大于原始数组的长度时，此类索引才会存在。所得数组与原始数组具有完全相同的类型。

会调用本地方法：System.arraycopy。

输出：

[1, 2, 1]
[1, 2, 1, 5, 8, 6, 10, 53, 44, 72, 33, 0, 0, 0, 0]

asList

该方法转换成的List对象是一个Arrays的内部类，实际使用时要：new java.util.ArrayList<>(Arrays.asList(nums)).

类似的还有java.util.ArrayList的subList方法。
