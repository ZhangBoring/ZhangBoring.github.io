Create a Resource Controller

在使用 Spring 来创建 RESTful Web 服务的方式中， HTTP 请求由 controller 处理。 这些组件由 @RestController 标识：

  • @GetMapping 注解确保对 /greeting 的 HTTP Get请求会被映射到 greeting() 方法。

  • @RequestParam 注解将查询字符串中的 name 参数绑定到 greeting() 方法的name参数中。

  • @RestController 注解标记了该类是一个返回域对象而不是视图的 controller。是 @Controller 和 @RequestBody 的简写。

  • Greeting对象必须转换成 JSON 才能响应回客户端。不过由于有 Spring 的 HTTP 消息转换器的支持，所以无需手动配置。因为 Jackson2 被依赖添加到 classpath 中，所以 Spring 会自动选择使用MappingJackson2HttpMessageConverter将Greeting实例转换为 JSON。

  • @SpringBootApplication 是一个方便的注解，它包括了一下注解:

    • @Configuration: 将类标记为程序上下文 bean 的定义源。
    • @EnableAutoConfiguration: 告诉 SpringBoot 根据 classpath 设置 、其他 bean 和 各种属性配置开始添加 bean。例如，如果 spring-webmvc在 classpath 里，该注解就会将应用标记为 web 程序，并激活一些关键行为例如设置DispatcherServlet。
    • @ComponentScan: 告诉 Spring 在当前包内查找其他 component 、 configuration 、 service 、 controller。
